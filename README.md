# ApplauseML — Release Risk Intelligence

Predictive ML tool that helps engineering teams assess bug risk before pushing a release candidate or site to production.

## What it does

| View | Description |
|------|-------------|
| **Risk Dashboard** | Always-on ranked view of historically risky components, platforms, and environments |
| **Release Predictor** | Form-driven prediction: enter release details, get H/C bug probability + component breakdown |
| **Monthly Digest** | Historical trend analysis refreshed monthly |

## Setup

```bash
# 1. Install dependencies
pip install -r requirements.txt

# 2. Train the model (run once, then monthly to refresh)
python model/train.py

# 3. Launch the app
streamlit run app/Home.py
```

## Project structure

```
ApplauseML/
├── data/               # Excel training data (bug details, test cycles, etc.)
├── model/
│   ├── train.py        # Training pipeline — run to produce artifacts
│   ├── predict.py      # Inference module used by the app
│   └── artifacts/      # Saved model files (generated by train.py)
├── app/
│   ├── Home.py         # Streamlit entry point
│   └── pages/
│       ├── 1_Risk_Dashboard.py
│       ├── 2_Release_Predictor.py
│       └── 3_Monthly_Digest.py
├── config.py           # Shared paths and feature definitions
└── requirements.txt
```

## Refreshing the model

Retrain monthly (or whenever new data is available):

```bash
python model/train.py
```

Artifacts are saved to `model/artifacts/`. Restart the Streamlit app to pick up the new model.

## Adding new data sources

To connect Jira, TestRail, ADO, or other bug tracking systems, add an importer in `data/` that outputs Excel files matching the existing schema, then rerun `model/train.py`.
